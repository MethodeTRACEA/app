# SESSION TRACÉA — Expérience utilisateur consolidée

_Date : 2026-04-11_

## Objectif global
Créer une régulation émotionnelle rapide, incarnée et accessible, même en état de surcharge.

Objectif utilisateur :
- quitter un peu le mental
- revenir au corps
- réduire un peu la charge
- faire émerger ce qui aide maintenant
- poser un geste faisable ou garder un repère concret

## Philosophie
TRACÉA n’est pas :
- un espace d’analyse
- une thérapie
- un journal
- un test de réussite émotionnelle

TRACÉA est :
- un passage
- une descente vers le corps
- une aide à retrouver de la marge de manœuvre
- un outil de récupération émotionnelle rapide

## Structure globale
TRACÉA propose 2 expériences.

### 1. Traversée courte — V1
Pour :
- état très activé
- confusion
- urgence de simplicité

Logique :
- entrée immédiate
- ressenti
- corps
- ancrage
- feedback
- action
- synthèse sobre

### 2. Parcours long — V1 quasi stabilisée
Pour :
- état plus modéré
- plus de disponibilité mentale
- besoin de traversée guidée plus complète

Logique :
- intro claire
- Traverser
- Reconnaître
- Ancrer
- Écouter
- Émerger
- Aligner
- synthèse finale IA courte

## Règles UX non négociables
- 1 écran = 1 action
- aucune surcharge cognitive
- langage concret
- jamais de jargon psychologique
- jamais de pression de performance
- toujours permettre une version plus simple
- toujours laisser une sortie douce
- UX avant IA

## Traversée courte — état actuel
Le flow court est considéré comme suffisamment bon pour sortir de la phase de patchs lourds.

### Points validés
- entrée comprise immédiatement
- flow rapide
- branches adaptatives cohérentes
- synthèse finale fidèle
- sortie sobre

### Rôle produit
- décrue rapide
- voie principale si forte activation
- porte discrète vers le parcours long

## Parcours long — état actuel
Le flow long a été fortement resserré.

### Points désormais acquis
- entrée plus claire
- contexte explicite
- Traverser scindé en sous-phases
- Reconnaître avec validation explicite
- Ancrer plus contrôlable
- Écouter allégé
- Émerger lisible
- Aligner simplifié
- écran final plus crédible
- micro-réception légère présente

### Points restant à observer en usage réel
- justesse des wording fins
- valeur perçue de la synthèse finale
- sensation d’accompagnement sans surcharge

## Place de l’IA
### Flow court
- quasi absente
- aucune analyse
- aucun écran IA autonome

### Flow long
- IA visible uniquement dans la synthèse finale
- rôle : relier, reformuler sobrement, retenir le mouvement
- jamais d’interprétation

## Couche produit active
Tracking minimal désormais présent :
- session_start
- step_complete
- session_end

Objectif : passer d’un pilotage intuitif à un pilotage fondé sur le réel.
