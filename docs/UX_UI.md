# UX / UI — état actuel consolidé

_Date : 2026-04-11_

## Objectif UX
Créer une expérience de régulation émotionnelle :
- simple
- incarnée
- centrée sur le corps
- utilisable même en activation
- orientée vers un prochain pas faisable, pas vers une preuve d’amélioration

## Architecture produit
TRACÉA repose sur 2 parcours distincts.

### 1. Traversée courte — V1 gelée
Usage : activation forte, faible tolérance cognitive.

Caractéristiques :
- entrée immédiate
- peu de texte
- peu de choix visibles
- auto-advance
- ancrage prioritaire
- IA très contenue

### 2. Parcours long — V1 quasi stabilisée
Usage : activation plus modérée, plus de bande passante.

Caractéristiques :
- progression explicite
- 6 étapes maintenues
- plus de structuration
- micro-réception légère
- synthèse finale IA courte

## Principes fondamentaux
- 1 écran = 1 action
- 1 question = 1 idée
- priorité au corps
- pas d’analyse
- pas de réflexion lourde
- toujours permettre une version plus simple
- toujours garder une sortie douce
- UX avant IA

## Traversée courte — statut
- stabilisée
- testée sur profils très activé / confus / modéré disponible
- considérée comme base de recette

## Parcours long — statut
- fortement simplifié
- désormais fluide du début à la fin
- détails de finition restants à arbitrer via usage réel, pas intuition

## Place actuelle de l’IA
### Flow court
- quasi absente
- jamais autonome
- jamais analytique

### Flow long
- visible seulement en synthèse finale
- sobre, structurante, non interprétative

## Point stratégique UX actuel
Le travail n’est plus de refondre les flows.
Le travail est maintenant de :
- mesurer
- lire les comportements réels
- corriger uniquement les frictions prouvées
